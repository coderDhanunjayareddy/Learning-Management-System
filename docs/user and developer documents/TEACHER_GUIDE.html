Question Form - Section 3: Marking Scheme

Set Up Marks:

Positive Marks

How many points for a correct answer?
  • Standard: 1 mark
  • For important questions: 2-4 marks
  • For problem-solving: 5+ marks
💡 Tip: Use higher marks for harder questions

Negative Marks (Optional)

Deduct marks for wrong answers (discourages guessing)
  • Leave as 0 for no negative marking
  • Use -1/4 of positive: If +4, then -1
  • Common pattern: +1 correct, -0.25 wrong
💡 Tip: Use negative marking for easier questions, not hard ones

Question Form - Section 4: Answer Options (Type Specific)

[SCREENSHOT 5: Answer Options Section - MCQ Single]
Show: Option A, B, C, D input fields with radio buttons, one selected as correct

For Multiple Choice (Single Answer)

Enter Options and Select Correct One:
  1. You'll see fields for Option A, B, C, D, etc.
  2. Type each option in its field
  3. Use rich text formatting for each option if needed
  4. Click the radio button (○) next to the correct option
  5. Radio button will fill in (●) showing it's correct
Example Setup: 
Option A: The mitochondria is the powerhouse of the cell
Option B: The ribosome is the powerhouse of the cell
Option C: ● (CORRECT) The nucleus is the control center of the cell
Option D: The endoplasmic reticulum is the powerhouse of the cell

For Multiple Choice (Multiple Answers)

[SCREENSHOT 6: Answer Options Section - MCQ Multiple]
Show: Option A-E with checkboxes, multiple checked as correct
Enter Options and Check Correct Ones:
  1. Type each option in its field
  2. Check the checkbox (☐) next to EACH correct option
  3. Checkboxes will show checkmarks (☑) for correct answers
  4. You can check multiple boxes
⚠️ Important Note: For multiple answer questions, grading is ALL-OR-NOTHING. Students must select EVERY correct option and NO incorrect ones to get marks. There's no partial credit.

For True/False

Select the Correct Answer:
  1. Write your statement in the question field
  2. Click either the "TRUE" or "FALSE" button
  3. The selected button highlights

For Numerical Answers

[SCREENSHOT 7: Numerical Answer Setup]
Show: Expected Value field, Tolerance field, help text
Enter Expected Answer and Tolerance Range:
  1. Enter the Expected Value (correct answer): e.g., 9.8
  2. Enter the Tolerance (acceptable range): e.g., 0.1
  3. System accepts answers within ±tolerance
Example - Physics Calculation:
  • Question: If gravity g = 9.8 m/s², calculate acceleration
  • Expected Value: 9.8
  • Tolerance: 0.1
  • Accepted Range: 9.7 to 9.9
  • Student answers 9.75: ✓ Correct (within range)
  • Student answers 9.5: ✗ Wrong (outside range)

Question Form - Section 5: Solution & Explanation

Add Solution (Highly Recommended):
  1. Scroll to "Solution/Explanation" section
  2. Click "Add Solution"
  3. A rich text editor opens
  4. Type detailed explanation of correct answer
  5. Include worked examples for math questions
💡 Good Solutions Include:
  • ✓ Why the correct answer is right
  • ✓ Why other options are wrong
  • ✓ Key concepts being tested
  • ✓ Step-by-step working for calculations
  • ✓ Real-world examples or analogies
  • ✓ Common mistakes to avoid

Saving Your Question

Step 3: Save or Submit Question
  1. Scroll to the bottom of the form
  2. You'll see two buttons: "Save as Draft" and "Submit for Approval"

Save as Draft

Use when:
  • Question is not yet complete
  • You want to revise it later
  • You want to test it before submitting
What happens:
  • Question saved with status "Draft"
  • Only you can see it
  • You can edit anytime
  • Cannot be used in exams yet

Submit for Approval

Use when:
  • Question is complete and correct
  • You want admin to review it
  • You're ready to use it in exams
What happens:
  • Question status becomes "Pending Approval"
  • Admin checks correctness, clarity, formatting
  • Admin approves or provides feedback
  • Once approved, you can use in exams
  • Typical approval time: 24 hours

3. Question Management & Workflow

Understanding Question Status

[SCREENSHOT 8: Question List with Status Badges]
Show: List of questions with colored status badges (Draft, Pending, Approved, Rejected)

Question Status Workflow

1️⃣ DRAFT
  • Status: Personal work in progress
  • Who sees it: Only you
  • Can edit: Yes, freely
  • Can use in exam: No
  • Actions: Edit, Delete, Submit for Approval
2️⃣ PENDING APPROVAL
  • Status: Awaiting admin review
  • Who sees it: You and admin only
  • Can edit: No, locked while pending
  • Can use in exam: No
  • Wait time: Typically 24 hours
3️⃣ APPROVED ✓
  • Status: Passed review, ready to use
  • Who sees it: You and all teachers
  • Can edit: Yes (re-enters pending workflow)
  • Can use in exam: Yes!
  • Actions: Edit, Use in Exam, Archive
4️⃣ REJECTED ✗
  • Status: Admin found issues
  • Who sees it: You and admin
  • Admin feedback: Included in rejection message
  • Can edit: Yes
  • Actions: Edit and Resubmit

Editing Questions

To Edit a Question:
  1. Find question in your list
  2. Click the "Edit" button (pencil icon) or click question name
  3. Make your changes
  4. Click "Update" to save
💡 What Happens When You Edit:
  • Draft questions: Updated immediately
  • Approved questions: Changes put back to "Pending Approval"
  • Already-used questions: Can edit (affects future attempts, not past)

Responding to Admin Feedback

If Your Question is Rejected:
  1. Check the rejection message for feedback
  2. Open the question in edit mode
  3. Make the requested changes
  4. Optionally add a note: "Fixed formatting issue" or "Corrected answer"
  5. Click "Submit for Approval" again
  6. Wait for re-review
Example Rejection Feedback:
"Good question, but the image is blurry. Please replace with higher resolution image. Also add a solution explanation so students understand the concept."

Archiving Questions

Archive Questions You No Longer Use:
  1. Find question in list
  2. Click menu icon (three dots)
  3. Select "Archive"
  4. Question is soft-deleted (not permanently removed)
  5. Won't appear in normal searches
  6. Can restore from archive if needed

Exam Form - Section 1: Basic Information

Exam Title (Required)

Good exam titles:
  • "Biology Midterm - Chapters 5-7"
  • "Mathematics Quiz 1 - Quadratic Equations"
  • "Physics Final Exam - Mechanics & Thermodynamics"

Description (Optional but Recommended)

Include:
  • Topics covered
  • Format (multiple choice, numerical, etc.)
  • Special instructions ("Show working", "No calculator", etc.)
  • Passing marks requirement
Example:
"This exam covers quadratic equations chapters. Questions are MCQ and numerical. Show working for numerical questions. 75+ marks required to pass. 1 hour duration."

Duration (Required)

💡 Time Planning:
  • Allocate 1-2 minutes per question minimum
  • 30 questions = 30-60 minutes
  • Add 5-10 minutes buffer for review
  • Consider question complexity

Total Marks (Required)

Standard marking:
  • Simple exam: 100 marks
  • Comprehensive: 150+ marks
  • Mark distribution: Should match marks in questions
Note: System will calculate total from questions you add

Passing Score (Optional)

Common passing marks:
  • 40% (40 marks out of 100)
  • 50% (50 marks out of 100)
  • 60% (60 marks out of 100)

Maximum Attempts

Option Meaning Use Case
1 Student gets only 1 try High stakes exam, final test
2-3 Student can retake 2-3 times Quiz, formative assessment
Unlimited Student can attempt anytime Practice tests, learning exercises

Shuffle Questions

Enable Shuffle Questions when:
  • ✓ Multiple students taking exam (reduces cheating)
  • ✓ You want each student to see different order
Disable when:
  • ✗ Questions are intentionally ordered (easy to hard)
  • ✗ You want consistency for fairness

Shuffle Options Within Questions

Enable when:
  • ✓ Multiple choice with different correct answers
  • ✓ Want to prevent position-based cheating (always option C, etc.)

Show Answers After Submission

Enable when:
  • ✓ Formative quiz (learning focused)
  • ✓ Practice exam
  • ✓ Students should see correct answers
Disable when:
  • ✗ Summative exam (grades focused)
  • ✗ You want to keep answers secret

Exam Form - Section 3: Availability Window

Set When Exam is Available:
  1. Click Start Date/Time - when exam becomes available
  2. Click End Date/Time - when exam closes
  3. Students can only start exams between these times
  4. If exam duration extends past end time, it auto-submits
Example Timeline:
  • Start: Monday 9:00 AM
  • End: Friday 5:00 PM
  • Duration: 60 minutes
  • Result: Students can start exam anytime Mon-Fri 9am-5pm. They have 60 min from when they start.

Reviewing Exam Before Publishing

Before Publishing, Check:
  1. ✓ All questions are approved (status = Approved)
  2. ✓ Question count and marks look right
  3. ✓ Questions are in logical order
  4. ✓ Total marks = Expected marks
  5. ✓ Difficulty distribution is balanced
  6. ✓ Duration is appropriate for question count
  7. ✓ Start/end dates are correct

Saving vs. Publishing

Step 4: Save or Publish Exam

Save as Draft

Use when:
  • Still adding/removing questions
  • Not ready for students yet
  • Need to review with colleague
Result:
  • Exam not visible to students
  • You can continue editing
  • Can add/remove questions freely

Publish Exam

Use when:
  • Exam is final and ready

6. Question Best Practices

Writing Effective Questions

DO:
  • ✓ Be clear and unambiguous
  • ✓ Test important concepts
  • ✓ Use appropriate language level
  • ✓ Provide realistic scenarios (where applicable)
  • ✓ Always include solutions
  • ✓ Ensure only one correct answer (for single MCQ)
  • ✓ Make incorrect options plausible
  • ✓ Review for spelling/grammar errors
DON'T:
  • ✗ Use trick questions or wordplay
  • ✗ Make questions too long (unless necessary)
  • ✗ Give away answers in incorrect options
  • ✗ Use "all of the above" or "none of the above" (ambiguous)
  • ✗ Reference other questions
  • ✗ Use outdated information
  • ✗ Have grammatical errors that confuse
  • ✗ Make options different lengths (easy to spot)

Incorrect Option Quality (Distractors)

Good vs. Bad Distractors:
  • Bad: "The moon orbits the sun" (obviously wrong)
  • Good: "The Earth orbits at 200 million km" (plausible but wrong)
  • Bad: "The radius is 157.08" (random number)
  • Good: "The radius is 15.7" (off by factor of 10, common mistake)

Balanced Exam Design

Difficulty Percentage Purpose
Easy 20% Build confidence, test basic knowledge
Medium 50% Test application, problem-solving
Hard 30% Test analysis, critical thinking

Negative Marking Strategy

When to use negative marks:
  • Easy questions: -1/4 of positive marks
  • Medium questions: -1/5 of positive marks (optional)
  • Hard questions: No negative (discourages correct guesses)
Example:
  • Easy: +4 marks, -1 for wrong
  • Medium: +3 marks, -0.75 for wrong
  • Hard: +3 marks, 0 for wrong

Key Metrics to Monitor

Metric What It Shows Action If Low
Pass Rate % of students who passed If <60%, exam may be too hard or students need more prep
Average Score Mean student score If <50%, review teaching or exam difficulty
Question Success Rate % who got each question right If <20%, question may be unclear or topic not taught

Using Data to Improve Teaching

Questions with low success rates:
  • Review the question - is it unclear?
  • Review teaching - did you cover this concept?
  • Revise the question for clarity
  • Provide additional instruction on concept
Class with low average:
  • Review exam difficulty - was it appropriately balanced?
  • Review student preparation - did they have time to study?
  • Provide remedial instruction on weak areas
  • Consider follow-up practice exam

9. Troubleshooting

Common Issues for Teachers

Problem: "Question marked as pending, can't use in exam"

Solution:
  1. Check question status - is it "Pending Approval"?
  2. If yes, contact admin - ask them to approve it
  3. Once approved, you can use it immediately
  4. If rejected, review feedback and resubmit
Problem: "I made an error in a question - students already took the exam"

Solutions:
  1. If error affects grading: Contact admin, they can manually adjust scores
  2. If error is just clarity: Update question explanation
  3. For future attempts: Edit question before students retake
  4. Apologize to students and provide extra credit if major error
Problem: "Student says their answer was correct but system marked it wrong"

Investigation steps:
  1. Review the student's answer in results
  2. Check if their answer matches the marked correct answer
  3. For numerical: Check tolerance range - is their answer within range?
  4. If you made an error in question setup, contact admin for manual adjustment
  5. If question is truly ambiguous, award marks to all students who got similar answers

LaTeX & Formatting Issues

Problem: "LaTeX formula not rendering correctly"

Common mistakes:
  • Missing $ signs: Type $\sqrt{4}$ not \sqrt{4}
  • Mismatched brackets: $\frac{1}{2} correct, not $(1/2)$
  • Missing \ for special symbols: $\pi$ not $pi$
Solution:
  1. Test formula in question preview
  2. If still not working, try simpler version
  3. Add image as fallback if LaTeX fails

Getting Help

When to contact support:
  1. Technical issues: System won't let you save, form crashes
  2. Question approval: Ask admin why question was rejected
  3. Grading issues: Ask admin to investigate marking errors
  4. Account issues: Can't login, permissions missing
