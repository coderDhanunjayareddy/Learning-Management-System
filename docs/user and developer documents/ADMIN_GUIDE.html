Bulk User Import

To Import Multiple Users at Once:
  1. Click "IMPORT USERS" button
  2. A dialog opens
[SCREENSHOT 4: Import Users Dialog]
Show: Download template button, CSV upload area, import preview, Confirm button

CSV Format Requirements

CSV Headers (must be in this order): 
FirstName,LastName,Email,Role,SchoolId
John,Smith,john.smith@school.edu,Student,1
Jane,Doe,jane.doe@school.edu,Teacher,1
Robert,Johnson,robert@school.edu,Admin,1
Rules:
  • First row must be headers
  • Email must be unique
  • Role must be: Student, Teacher, or Admin
  • SchoolId must exist in system
  • UTF-8 encoding required

Import Process

Steps:
  1. Click "Download Template" to get CSV template
  2. Open template in Excel or Google Sheets
  3. Add your user data
  4. Save as CSV
  5. Return to import dialog
  6. Click "Upload CSV"
  7. System validates data
  8. Review import preview
  9. Click "Confirm Import"
  10. System creates all users

Editing Users

To Edit User Details:
  1. Find user in the list
  2. Click their name or the edit button
  3. Modify fields as needed
  4. Click "Update"
⚠️ Caution When Editing:
  • Changing email may break login access
  • Changing role affects system permissions immediately
  • Changing school affects content visibility

Managing User Status

Deactivating Users

To Deactivate a User (when they leave):
  1. Find user in list
  2. Click menu (three dots)
  3. Select "Deactivate"
  4. User can no longer login
  5. Their data is preserved
  6. Can be reactivated later if needed

Resetting Passwords

When User Forgets Password:
  1. Find user in list
  2. Click menu (three dots)
  3. Select "Reset Password"
  4. System generates temporary password
  5. Send to user via email
  6. User must change on first login

Viewing User Activity

To Track What a User Has Done:
  • Find user in list
  • Click "Activity" tab (if available)
  • See: Last login, exams taken, questions created, etc.

3. Question Approval Workflow

Accessing Pending Questions

Step 1: Navigate to Questions
  1. Click "Questions" in sidebar
  2. Click "PENDING APPROVAL" tab
  3. See all questions waiting for review
[SCREENSHOT 5: Pending Questions List]
Show: Question list with submitted date, teacher name, subject, question preview, action buttons

Reviewing Questions for Quality

Step 2: Review Each Question
  1. Click question to open review view
  2. Carefully examine all aspects
[SCREENSHOT 6: Question Review View]
Show: Full question text, options, correct answer highlighted, marks, solution, approve/reject buttons

Review Checklist

Content Quality:
  • ☐ Question text is clear and unambiguous
  • ☐ Correct answer is actually correct
  • ☐ All options are plausible
  • ☐ Solution explanation is helpful
  • ☐ Question tests appropriate concept level
Technical Quality:
  • ☐ No spelling or grammar errors
  • ☐ LaTeX formulas render correctly
  • ☐ Images are clear and relevant
  • ☐ Formatting is consistent
  • ☐ Subject/Chapter/Topic are correctly classified
Marking Scheme:
  • ☐ Marks are appropriate for difficulty
  • ☐ Negative marking (if used) is reasonable
  • ☐ Marks are clear in question

Approving Questions

When Quality is Good:
  1. Click "APPROVE" button
  2. Optionally add feedback: "Great question! Well structured."
  3. Question status becomes "Approved"
  4. Teacher receives approval notification
  5. Question available for use in exams
✓ Good Approval Feedback:
  • "Approved! Clear question with good distractors."
  • "Approved. Good use of LaTeX formatting."
  • "Approved. Excellent solution explanation."

Rejecting or Requesting Changes

When Quality Needs Improvement:
  1. Click "REQUEST CHANGES" or "REJECT"
  2. Write detailed feedback explaining issues
  3. Be specific: "What's wrong?" and "How to fix it?"
  4. Click "Send"
  5. Teacher receives feedback via email
✗ Poor Rejection Feedback:
"This question isn't good. Fix it."
✓ Good Rejection Feedback:
"The LaTeX formula in the question text isn't rendering correctly. Please use $...$ syntax. Also, the image is blurry - please replace with a clearer version. Finally, add a solution explanation so students understand the concept. Once revised, please resubmit."

Managing Approval Queue

💡 Best Practices:
  • ✓ Process approvals daily to avoid backlog
  • ✓ Set SLA: Approve/reject within 24 hours
  • ✓ Provide constructive feedback, not just rejection
  • ✓ Look for patterns in rejections (e.g., formatting issues) and educate teachers
  • ✓ Approve good questions immediately

Managing Schools

To Manage Schools:
  1. Click "Organizations" in sidebar
  2. Click "Schools" tab
[SCREENSHOT 7: Schools Management]

Creating a School

To Add a New School:
  1. Click "+ CREATE SCHOOL"
  2. Enter school details:
    • School Name (required)
    • Address
    • Contact Email
    • Contact Phone
    • Principal/Administrator Name

  1. Click on school name
  2. Go to "Members" tab
  3. Click "+ ADD MEMBER"
  4. Search and select users
  5. Assign role (Teacher, Admin, Student)

Managing Batches

What are Batches?

Batches are groups of students (like class sections). They help:

  • Organize students for better management
  • Assign specific exams to specific groups
  • Track progress by cohort
  • Manage access to content
To Create a Batch:
  1. Click "Organizations" → "Batches"
  2. Click "+ CREATE BATCH"
  3. Enter batch details:
    • Batch Name (e.g., "Class 10-A", "Batch Spring 2024")
    • School (which school)
    • Start Date
    • End Date
  4. Click "Create"

Adding Students to Batch

To Add Students to a Batch:
  1. Click on batch name
  2. Go to "Members" tab
  3. Click "+ ADD STUDENTS"
  4. Filter and select students
  5. Click "Add to Batch"
  6. Students now belong to this batch

5. Content & Entitlements

Content Packs

What are Content Packs?

Bundles of related questions, exams, or courses that can be distributed together to users or schools. Useful for:

  • Packaging curriculum materials
  • Distributing exam sets
  • Enabling specific content for specific users
To Create a Content Pack:
  1. Click "Content Packs" in sidebar
  2. Click "+ CREATE PACK"
  3. Enter pack details:
    • Pack Name
    • Description
  4. Click "Create"

Entitlements (Access Control)

Entitlements Control Who Can Access What

Used to:

  • Grant specific users/schools access to content packs
  • Manage subscriptions
  • Control feature access
To Create an Entitlement:
  1. Click "Entitlements" in sidebar
  2. Click "+ CREATE ENTITLEMENT"
  3. Specify:
    • Content Pack or Feature
    • User or School to grant access to
    • Valid From date
    • Valid To date (expiry)
  4. Click "Create"

8. Troubleshooting & Maintenance

Common Admin Issues

Problem: Pending questions accumulating (queue too long)

Solutions:
  1. Check if you have sufficient admin staff
  2. Set up approval SLA (e.g., 24-hour target)
  3. Conduct teacher training on question quality
  4. Consider delegating approvals to school admins
  5. Implement automated checks for common errors
Problem: User complaints of slow exam performance

Investigation Steps:
  1. Check server status/load
  2. Check network connectivity
  3. Look at exam with heavy images/videos
  4. Consider CDN for media delivery
  5. Contact infrastructure team for optimization
Problem: Data integrity - question changed after exam taken

Process:
  1. Check audit logs for modifications
  2. If teacher modified correct answer: Manual adjustment needed
  3. Award correct marks to affected students
  4. Communicate with teacher about data integrity
  5. Consider implementing question versioning

System Maintenance

Regular Maintenance Tasks

Daily:
  • ☐ Check admin dashboard for alerts
  • ☐ Monitor active exam count
  • ☐ Process pending question approvals
Weekly:
  • ☐ Review audit logs for suspicious activity
  • ☐ Check user activity/engagement
  • ☐ Backup database (if manual backup required)
Monthly:
  • ☐ Generate platform health report
  • ☐ Review user feedback/support tickets
  • ☐ Check for system updates/patches
  • ☐ Clean up archived content

💡 What to Monitor:

9. Security & Best Practices

Security Checklist

User Security:
  • ☐ Enforce strong password policy
  • ☐ Regular password resets
  • ☐ Deactivate inactive accounts
  • ☐ Monitor failed login attempts
  • ☐ Implement 2FA for admin accounts
Data Security:
  • ☐ Enable HTTPS everywhere
  • ☐ Regular database backups
  • ☐ Encrypt sensitive data
  • ☐ Audit log monitoring
  • ☐ GDPR compliance checks
Access Control:
  • ☐ Least privilege principle
  • ☐ Role-based access control
  • ☐ Regular permission audits
  • ☐ Segregate duties (don't approve own questions)

Data Privacy & Compliance

GDPR Compliance Considerations:
  • Maintain privacy policy
  • Get consent for data collection
  • Provide data export on request
  • Support account deletion
  • Maintain audit logs (24 months minimum)
  • Report data breaches within 72 hours

Incident Response

If You Suspect a Security Incident:
  1. Immediately disable affected accounts
  2. Check audit logs for suspicious activity
  3. Contact your security team/provider
  4. Document everything
  5. If data breach: Notify affected users within 72 hours
  6. Review and strengthen security after incident

Admin Best Practices

✓ DO:
  • ✓ Use strong, unique admin passwords
  • ✓ Enable 2FA for admin account
  • ✓ Regularly audit user permissions
  • ✓ Document admin decisions and changes
  • ✓ Keep admin team informed
  • ✓ Provide teacher training on question quality
  • ✓ Set clear SLAs (e.g., 24-hour approval)
  • ✓ Monitor system performance
✗ DON'T:
  • ✗ Share admin credentials
  • ✗ Login from public WiFi without VPN
  • ✗ Approve/reject questions without review
  • ✗ Modify student answers (except in error scenarios)
  • ✗ Keep old/unused admin accounts active
  • ✗ Ignore system alerts
  • ✗ Skip security updates
  • ✗ Store passwords in plain text
